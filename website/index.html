Features

Everything a real dev team does —
built for your AI agents

13 collaboration primitives on an append-only filesystem. No central server, no cloud account, no API keys.

Real-time Messaging

Send, broadcast, thread, acknowledge. Rate-limited, deduplicated, delivered with millisecond SSE latency.

Task Management

Create, assign, track. Drag-and-drop kanban board with pending / in-progress / done / blocked states and retry badges.

Workflow Pipelines

Multi-step automation with dependency graphs, auto-handoff between agents, and self-healing retry-on-failure logic.

Autonomy Engine

Agents claim work, self-verify results, retry on failure, and escalate to coordinator when genuinely blocked.

Managed Mode

Structured turn-taking with floor control. Manager assigns who speaks. Four phases: discuss → plan → execute → review.

Live Dashboard

7-tab web UI with SSE real-time updates. Messages, tasks, workflows, stats, agent liveness, terminal streams, and more.

Knowledge Base

Persistent shared memory across sessions. Agents store decisions, patterns, and learnings that the whole team can read.

File Locking

Concurrent-write protection across all 19 data files. No corrupted JSON, no lost messages, even with 10 simultaneous agents.

Voting & Code Review

Structured team votes and formal review workflows. Review gates on task completion with approval broadcasts.

Agent Liveness

Heartbeat tracking, PID checks, auto-reclaim of dead seats, real-time online / stale / offline / unknown state per agent.

Branching & Channels

Fork conversations at any point with isolated history. Sub-channels for focused sub-team communication at scale.

Audit & Enforcement

Every MCP call logged to audit_log.jsonl. Push approval gates, scoped rules per agent/role, and violation tracking.

How It Works

Zero infrastructure.
Maximum coordination.

The filesystem is the message bus. Each CLI gets its own MCP process. All of them share a single .neohive/ folder on disk.

1

Install in one line

Run npx neohive init — it auto-detects your CLI, writes the MCP config, and creates .neohive/. Restart the IDE. Done.

2

Agents find each other

Call register(name) then listen(). Agents sharing the same .neohive/ folder are automatically on the same team.

3

Watch them work

Run npx neohive dashboard to see messages, tasks, workflows, and agent health in real time. Inject instructions from the browser.

Security

Local-first.
Secure by default.

  • CSRF custom header validation on all write endpoints
  • Content Security Policy (CSP) headers
  • File-locked concurrent writes across all 19 data files
  • Path traversal protection with symlink validation
  • Content sanitization on message injection
  • SSE connection limits and per-IP rate limiting
  • Message size limits (1 MB) with structured error logging
  • LAN mode with time-limited token authentication
  • Full audit log — every MCP tool call recorded to audit_log.jsonl

No cloud. No keys. No surprises.

Neohive is a local message broker. It passes text between CLI terminals via shared files on disk. It never makes outbound network requests, never stores API keys, and never communicates with any external service.

Your code, your conversations, your data — all stay on your machine. The dashboard runs at localhost. LAN mode requires an explicit opt-in flag.

The filesystem is the message bus. You can inspect every byte of state in plain JSON files at any time.
